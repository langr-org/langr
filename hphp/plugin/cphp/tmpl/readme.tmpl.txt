关于模板文件的修改说明:

在 "{$" "}" 之间的变数名是由程式自动填写的, 在修改或新建模板时
应该不要改动 "{$var}" 的名字, 你最好要能明白变数 "{$var}" 的含
意及它可能出现的值, 这对构建一个漂亮的模板很重要, 比喻说, 有一
个模板变数: {$webSiteTitle}, 它表示这个 html 页面的标题. 如果
你把它放在 <a href="{$webSiteTitle}">/</a> 显然不是太好的, 不
过你可以在一个模板中多次使用一个变数, 像上面的使用方法肯定是不
会出错的. 当然, 如果必要, 你可以自己定义一些变数名, 并通知你的
程序员按照你的要求去填写它, 一切都会以你的意见为重哦.

在模板文件中, 你能常看到一个变数名就能写出一整个可变长度表格来 
,这种变数就会涉及到 子模板文件. 比喻说, 在模板里有一个变量 
{$subTop}, 它能生成一个关于今天人气最多的相簿表格来. 其实它会
用到一个子模板文件, 修改相应的子模板文件就能改变 {$subTop} 的
显示方式. 子模板文件一般和它的母模板文件名相近, 如果你修改的某
个模板文件名如: index.tpl.php, 那么它用到的子模板文件名可能会
是这样的 index.list01.tpl.php, 中间的 list01 就是这个子模板名.
子模板中包括在两个 "<!--TMPL:Line-->" 之间的内容一般表示这个表
格的一行里的内容, 程序会根据这一行的内容来生成更多的行. 而且子
模板还可以包括多段以 "<!--TMPL:Line-->" 分割的内容, 只要有必要. 

还有一种你可能看到的是像下面一样的代码:
<?
$this->loadTmplate(TEMPLATE_PATH."public/head.tpl.php");
?>
它的意思是装载另一个模板文件. 上面的代码就表示将模板目录中的一
个 "public/" 目录下面的 "head.tpl.php" 文件装载到这里, 你可以把
它理解为: 用模板目录文件 public/head.tpl.php 中的内容来替换掉上
面你可能不太喜欢的三行代码. 一般的情况下, 你可能会把一些很多模
板文件都会用到的内容放到一个公共的模板文件里面, 如同一个站点里
每个 html 页面都可能会有相同的头部和尾部, 这时你把它们的头和尾
都放到公共模板文件中, 并用上面三行代码来代替它们. 

对于模板中表单的 name 和 value 值应该是不能随便改动的, 最起码你
得和你的程序员商量一下, 以免产生些不必要的麻烦. 
如果你要在模板文件中加入链接到本地的超链接, 你可以试着用用这个
方式: 
<a href="<?=url('index.php?module=mod&other=other')?>" title=
"title">link</a>
url()函数中的路径以项目目录为当前目录, 这样你的链接可能会看起来
更漂亮. 当然, 这并不是必须的. 而且这需要你的程序员配置了正确的
服务器 Rewrite 功能.
